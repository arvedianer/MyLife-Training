## M2 — Sprint 2: App Completeness & Polish 🚧 CURRENT

### Phase 1 — Unilateral Rework + Equipment Swap [ ]
Fix Unilateral komplett (1 Gewicht, L/R Wdhl, breitere Inputs, Seitenvergleich).
Equipment-Swap-Logik: Übungsvariation bei Equipment-Wechsel vorschlagen.

### Phase 2 — Übungsdatenbank Erweiterung [ ]
Mehr Übungen (100+ gesamt). ExRx/Wger-Daten integrieren.
Übungs-GIFs/Animationen via öffentliche API oder statische Assets.
Equipment-Variationen als strukturierte Daten.

### Phase 3 — Splits Rework [ ]
Intelligentere Plan-Generierung (Onboarding-Daten nutzen).
Mehr & bessere Pläne (PPL, Upper/Lower, Fullbody, Bro-Split, etc.).
Split-Tab zeigt Übungen direkt mit Sets/Reps.
Split-Specs: Dauer, Schwierigkeitsgrad, Zielgruppe.

### Phase 4 — Stats Rework [ ]
Athlete Score → All-Time Best + Weekly Score.
Kraft-Vergleiche: klare Darstellung (kein "Top 1% / 99%" nebeneinander).
Kraftentwicklung als Chart (progressive overload sichtbar).
Trainingstage-Section entfernen.
Aktivitätskalender überarbeiten.
Ausdauer aus Score-Berechnung raus, Stärke-Dimension verbessern.

### Phase 5 — Coach Arved UX + Forum [ ]
Quick Replies nach Antwort ausblenden.
Bessere Coach-Einstiegspunkte (was kann ich mit Coach machen?).
Coach Arved als KI-Teilnehmer im Forum.

### Phase 6 — Onboarding Done-Screen + Tutorial [ ]
Done-Screen neu designen (schöner, mehr Kontext).
Nach Onboarding: "Tutorial starten?" Prompt.
Tour/Tutorial komplett überarbeiten (animierter, natürlicher, funktionierend).

### Phase 7 — Active Workout + Summary Polish [ ]
Mikrofon weniger prominent (optional/versteckt).
Workout Summary Rework (Score-Darstellung, Layout).
